Project Overview 
================

What this project is
--------------------
This is a NestJS + Prisma chat application with:
- email/password auth
- 1-to-1 chat requests
- group chat with invites
- file uploads for chat/avatar/group images
- Socket.IO realtime messaging
- push notifications
- Redis-backed support features

Main app areas
--------------
- src/auth: signup, login, OTP verification, password reset, JWT auth
- src/user: profile, settings, avatar upload, contact nickname/theme
- src/chat: direct chat, group chat, attachments, realtime gateway, backups
- src/mail: email sending for OTP and reset flows
- src/notifications: web push notifications
- src/prisma: Prisma database client
- src/redis: Redis connection service
- index.html: single-page frontend UI

Dependencies used and why
-------------------------
- @nestjs/common, @nestjs/core
  Base NestJS framework used across the whole backend.

- @nestjs/platform-express
  HTTP server integration for Nest and file upload interceptors.
  Used in chat/user controllers for uploads.

- @nestjs/websockets, @nestjs/platform-socket.io, socket.io
  Realtime chat, typing indicator, call signaling, read updates, message updates.
  Main usage: src/chat/chat.gateway.ts and index.html Socket.IO client.

- @nestjs/jwt
  JWT token signing and verification.
  Used in auth service, auth guard, and chat gateway socket auth.

- @prisma/client, prisma
  Database ORM and schema management.
  Used for users, messages, groups, requests, invites, subscriptions.
  Main usage: prisma/schema.prisma and src/prisma/prisma.service.ts.

- bcrypt
  Password hashing and password comparison.
  Used in src/auth/auth.service.ts and src/user/user.service.ts.

- class-validator
  Request DTO validation.
  Used in auth DTOs like register.dto.ts.

- multer
  Multipart file upload handling.
  Used through Nest FileInterceptor in user/chat controllers.

- nodemailer
  Sends email OTPs and reset emails.
  Used in src/mail/mail.service.ts.

- web-push
  Browser push notification sending with VAPID.
  Used in src/notifications/push-notification.service.ts.

- ioredis
  Redis client.
  Used in src/redis/redis.service.ts.

- helmet
  Basic HTTP security headers.
  Used in src/main.ts.

- express-rate-limit
  Request throttling.
  Config exists in src/main.ts.
  Note: currently commented out.

- express
  Static file serving for public/ and uploads/.
  Used in src/main.ts.

- @nestjs/swagger
  Only used for PartialType in src/auth/dto/update-auth.dto.ts.
  Swagger UI is not actively configured in main.ts right now.

Short flow summary
------------------
1. User registers/login through src/auth.
2. Prisma stores users/messages/groups in PostgreSQL.
3. index.html calls REST APIs for loading data and uploads.
4. Socket.IO handles live chat updates and typing.
5. Mail service sends OTP emails.
6. Push service sends browser notifications.
7. Redis supports chat-related background/cache behavior.

Backup note
-----------
- Live chat data stays in PostgreSQL.
- Scheduled backup archives are generated by src/chat/chat-backup.service.ts.
- If Cloudinary backup env vars are configured, backup JSON archives are uploaded to Cloudinary as raw files.
- If Cloudinary is not configured or upload fails, backups stay in the local backups/ folder.
