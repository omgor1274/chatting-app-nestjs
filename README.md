## Description

[Nest](https://github.com/nestjs/nest) framework TypeScript starter repository.

## Project setup

```bash
$ npm install
```

## Compile and run the project

```bash
# development
$ npm run start

# watch mode
$ npm run start:dev

# production mode
$ npm run start:prod
```

## Run tests

```bash
# unit tests
$ npm run test

# e2e tests
$ npm run test:e2e

# test coverage
$ npm run test:cov
```

## Deployment

This project can run on both Railway and Render.

Railway is already configured through `railway.json`. Render is now configured through `render.yaml`.

Use the same app code on both platforms, but decide whether you want:

1. Separate deployments with separate data.
2. Dual live deployments that share the same Postgres, Redis, and object storage.

If Railway and Render are both live against the same chat data:

- Set `ALLOWED_ORIGINS` to include both public domains.
- Use a shared Redis instance, otherwise realtime events can diverge across hosts.
- Use shared object storage such as Cloudflare R2 for attachments, otherwise uploaded files stay local to one host.
- Mount persistent disk storage at `/data` if you still keep any local uploads or backups.

Typical commands are:

```bash
npm run prisma:generate
npm run build
npm run prisma:migrate:deploy
npm run start:prod
```

Render blueprint notes:

- Build command: `npm ci && npm run prisma:generate && npm run build`
- Start command: `npm run prisma:migrate:deploy && npm run start:prod`
- Health check: `/health`
- Persistent disk mount: `/data`
